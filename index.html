Why use Vita?

Vita proves to have very easy syntax to learn so the learning curve and time is very short.

As if that is not all, thanks to dynamic typing, code lines in Vita are very short as there is no need for specifying datatypes of variables in your code.

Packaging is easier.

This is yet to be experimented, but as we know, interpreted languages do not compile to binaries and you need the whole interpreter to be packed with your source code. Still it is the way we have to do it here, but no worries the Vita runtime is very small in size and it is provided as a single executable allowing you easily embed it in your code.

No compiling.

Unlike other interpreted languages that compile code using JITs, Vita is now the real interpreter as it does not do anything like compiling during code execution but interprete line by line.

Simple code example.

This code only tells you the time after getting your name and if you do not enter any word in the console, you will have to retry.

									
import time

fn clock(name){
	print(`{name}, the time is {time.ctime()}.`)
}

fn main {
	name = readline("Enter name: ")

	if len(name)
		clock(name)
	else
		print("You never entered a name. Please retry.")
		main()
}
